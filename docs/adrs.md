# Architecture Decision Log

ADR-style decisions, ditambah seiring waktu. Indexed by number for grep convenience.

CLAUDE.md hanya menyimpan ADR title one-liners. Reasoning + tradeoff lengkap ada di sini.

---

- ADR-001: Build provider abstraction sendiri (bukan LiteLLM) untuk learning + kontrol
- ADR-002: Single inheritance untuk profile (bukan composition) untuk simplicity
- ADR-003: Cache key include profile_version (auto-invalidation)
- ADR-004: Streaming response postponed (sync batch sufficient untuk MVP)
- ADR-005: Glossary retrieval exact-match dulu, vector search di fase 2
- ADR-006: Provider abstraction pakai `Protocol` (structural), bukan `ABC` (nominal). Alasannya wrapper seperti `RetryingProvider` bisa satisfy interface tanpa inherit common base — DI-friendly dan test-friendly.
- ADR-007: Retry policy hand-rolled (bukan tenacity). Policy hidup di `RetryingProvider.translate` dekat dengan tempat keputusan dibuat, lebih mudah di-audit daripada config dict. Tradeoff: kehilangan fitur seperti jitter (boleh ditambah saat dibutuhkan).
- ADR-008: Cost accounting pakai `Decimal` (bukan float). Float bisa drift di multi-juta-token; `Decimal` punya jaminan associativity yang dibutuhkan untuk billing.
- ADR-009: `quality_mode` pakai `String` + `CHECK` constraint (bukan Postgres ENUM). Menambah value ke ENUM butuh `ALTER TYPE` DDL yang ribet di migration; CHECK cuma butuh column-level migration.
- ADR-010: Test DB pakai `NullPool` (bukan pool default). pytest-asyncio bikin event loop baru per test; pooled asyncpg connection punya event-loop affinity sehingga reuse antar test bikin "another operation is in progress" error. NullPool ensures fresh connection per session.
- ADR-011: ProfileVersion snapshot disimpan sebagai JSONB `dict[str, Any]` (bukan typed schema). Snapshot adalah audit log — kalau ProfileRead schema berubah, snapshot lama tetap harus bisa dibaca tanpa migration. Looseness intentional.
- ADR-012: Cache key truncated ke 32 hex chars (128 bits entropy). Cukup unique untuk lifetime cache (collision probability functionally zero) tapi separuh storage di Redis dibanding 64-char full SHA-256 digest. Joining pakai ASCII unit separator (`\x1f`) supaya "ab"+"c" tidak collide dengan "a"+"bc".
- ADR-013: Cache layer graceful degradation: SEMUA `RedisError` ditelan, `get` return None, `set` no-op, `delete` no-op. Cache adalah perf primitive, bukan correctness. Trade-off: Redis misconfig terlihat sama dengan Redis down — ops sees warnings di log dan `health_check` endpoint reports state.
- ADR-014: Jinja prompt template di-render full lalu di-pass sebagai `system_prompt_override` ke provider. Provider tetap bangun user message simple ("Translate from X to Y... Text: ...") — text muncul dua kali (system + user), redundansi minor tapi tidak butuh modifikasi ClaudeProvider contract Phase 2.
- ADR-015: NFC unicode normalisation di `validate_and_normalize`. Tanpa ini, "café" (NFC single codepoint) dan "café" (NFD = "e" + combining acute) menghasilkan cache key berbeda — frustrating bug yang invisible kecuali someone pastes from a Mac.
- ADR-016: Cache + provider sebagai `@lru_cache` singletons di `src/api/dependencies.py`, resolver + pipeline + repo per-request. Singleton untuk yang punya pool (Redis connection pool, Anthropic SDK client); per-request untuk yang butuh DB session. Single instance per process menghindari double connection pool / SDK client init cost.
- ADR-017: Profile soft-delete (flip `is_active=False`, bump version) bukan hard-delete. Cache entries dengan version lama natural fall-off via TTL; future audit/rollback masih bisa baca row. Hard delete bakal orphan glossary + examples FK-cascade-style.
- ADR-018: API test fixture override `session.commit` jadi no-op (treat as flush). Routes call `repo._session.commit()` per write (correct in prod); di test ini akan persist past the rollback fixture dan leak ke test berikutnya. Patching commit jaga rollback discipline.
- ADR-019: API errors selalu return `ErrorResponse` envelope `{error_code, detail, trace_id}`. Machine-readable `error_code` (e.g. `rate_limited`, `profile_not_found`) supaya SDK clients bisa bercabang tanpa parsing free-text detail. Trace_id bikin error reports lebih mudah dikorelasikan ke server logs.
- ADR-020: Eval metrics dinormalisasi ke `[0.0, 1.0]` regardless of native scale. sacrebleu chrF natively 0..100; di harness kita bagi 100. Tujuannya: aggregate per-metric stats (mean/p50/p95) bisa di-summed/averaged tanpa code yang tahu metric-specific scale.
- ADR-021: `GlossaryComplianceMetric` adalah adapter tipis ke `src.pipeline.compliance.compute_glossary_compliance`, BUKAN re-implementasi. Eval score harus identical dengan runtime `glossary_compliance` field. Reimplement bakal silently drift — debug "kenapa eval bilang 0.8 tapi prod log bilang 0.85" jauh lebih mahal dibanding satu function call.
- ADR-022: Eval runner prompts for confirmation before spending real money (`--yes` opt-out). Default tradition: capabilities yang membakar uang/credentials/destruction butuh explicit consent. Cost preview pakai `estimate_cost` per entry (worst case, cache hits will reduce actual spend).
- ADR-023: SDK ditulis sebagai standalone ES2020 JS (no build step, no bundler). Drop-in `<script src=...>`. Trade-off: tidak bisa pakai modules/imports → semuanya di satu file dalam IIFE pattern. Tapi user (operator yang nyalain SDK di landing page) tidak butuh npm + esbuild setup, dan satu file lebih mudah di-cache di CDN.
- ADR-024: SDK pakai TreeWalker untuk text nodes, BUKAN innerHTML rewrite. Inline markup (`<a>`, `<strong>`) di tengah paragraf tidak boleh di-flatten. TreeWalker yields raw text nodes; kita ganti `nodeValue` in place. Trade-off: kalau translation memerlukan reordering markup, kita gak bisa handle — tapi untuk MVP single-language reordering jarang dibutuhkan.
- ADR-025: SDK client cache pakai 2-tier (LRU memory + localStorage). Profile version fetched once di init lalu di-fold ke cache key, sehingga server-side bump otomatis invalidate client cache. localStorage failures (quota, private mode) silent fallthrough — page harus tetap jalan kalau browser refuse persistent storage.
- ADR-026: Translation log menyimpan `source_text` & `translated_text` full plaintext. PII trade-off diterima untuk MVP (single-tenant, operator internal); mitigasi reaktif berupa future redaction flag + dashboard access control kalau pelanggan eksternal onboard.
- ADR-027: `record_log` swallows semua exceptions di pipeline's `finally` block (broad `except Exception`); `log_id` nullable di response untuk signal write failure. record_log MUST NEVER raise — a raise di finally bakal mask original pipeline exception. Cache-pattern degradation (ADR-013) di-extend ke log layer — translation API tetap healthy meski log store unhealthy.
- ADR-028: `error_detail` sanitization minimal via regex (`sk-ant-…`, `Bearer …` patterns), expand reactively bukan upfront. Over-engineering token-pattern detection sebelum pattern muncul di logs nyata = bikeshedding.
- ADR-029: Concurrent `session.flush()` di batch endpoint diserialize via process-level `asyncio.Lock` per-session (`_SESSION_LOCKS` di `src/translation_logs/repository.py`). SQLAlchemy AsyncSession tidak allow concurrent flushes; `asyncio.gather` di batch route bakal trigger "Session is already flushing" errors yang silently di-swallow oleh record_log → log rows hilang. Lock serializes hanya flush call (not provider calls), preserves batch parallelism untuk heavy LLM work. Trade-off: dict grows unbounded per process (bounded by alive-request count, acceptable for MVP); cleaner future fix adalah per-batch-item session.
- ADR-030: `AitegrityProductSpec` di seed script defined inline sebagai `@dataclass` list, bukan eksternal YAML/JSON. Code IS data: type-checked by mypy, no parsing layer, refactor-safe di IDE. Trade-off: non-coder operators harus edit Python, tapi mereka populate ongoing data via Streamlit UI anyway sehingga external-file editing path tidak benar-benar dipakai.
- ADR-031: Agent failures soft-fail by default — `LangDetectAgent`-style agents catch their own exceptions and emit `status="failed"` activities; only `TranslateAgent` (the primary value) propagates errors. Orchestrator's `_safe_run` captures translate exceptions along with sibling activities, then re-raises after the group completes so partial state lands in the log row. Extends graceful-degradation pattern from ADR-013 (cache) and ADR-027 (record_log) to the agentic layer.
- ADR-032: `AgenticActivity.result` typed as `dict[str, Any]` (JSONB-serialized), not typed per agent. Shape varies per agent type (lang_detect: `{detected_lang, confidence}`; translate: `{translation, stop_reason}`). Union with discriminator complicates Pydantic + JSONB serialization for low value; future agents extend without schema migration.
- ADR-033: Lang detection backend = Claude Haiku (LLM). Authentic agentic narrative for stakeholder demos; cost overhead negligible (~$0.0001/call vs. $0.001-0.005 main translate); reuses ClaudeProvider abstraction. Frontend langdetect (Streamlit typing-detection UX added with prompt_applied) retained at a different layer for a different purpose (proactive type-time vs. confirmatory post-request).
- ADR-039: Sub-proyek H discarded (migrations 005-007 deleted, code stashed). Sub-proyek I starts fresh from migration 004. Reason: H model was conceptually rejected by user (denormalized rename instead of true junction); clean restart cheaper than incremental fix on rejected foundation.
- ADR-040: Custom ID format `{prefix}-{8hex}-{4hex}` for all PKs in sub-proyek I (country, company, department, position, service, tenant, tenant_profile, tenant_prompts). 48 bits entropy (lifetime collision-safe at MVP scale per ADR-012 precedent). More readable than full UUID for log greps + admin operations; fits VARCHAR(30). Helper in `src/db/ids.py`.
- ADR-041: Tenant = junction of (country, company, department) with built-in auth columns (api_key_hash, jwt_active_token, jwt_refreshed_at). Composite UNIQUE on the three FK columns. 57 rows seeded (3 companies × 19 departments × 1 country per company). Avoids a separate auth table — every operator IS a tenant for the MVP scope.
- ADR-042: Tenant_profile = nested junction of (position, service) attached to a tenant. `prompt_applied` is an array of `prompt_id` strings (variable-length config per profile) so operators can mix-and-match the 3 seeded prompts (lang_detect_input, lang_detect_output, translate) without altering schema.
- ADR-043: Tone, target_audience, glossary, style_examples all moved to `service` (not tenant_profile). Reflects that these are properties of the service offering, not the operator (company/department/position) running it. Reduces data duplication; future service-level prompt customisation more natural.
- ADR-044: Position has `department_id NOT NULL FK ON DELETE CASCADE`. DB-enforces the org chart: a position name like "Analyst" can recur across departments only as distinct rows because the unique constraint is on `(position_name, department_id)`.
- ADR-045: API key argon2-hashed (passlib + argon2-cffi). Plaintext generated as `aitkey_<urlsafe-base64-32-bytes>` and returned ONCE during creation (seed stdout or admin endpoint response); never persisted plaintext. argon2 chosen over bcrypt for resistance to GPU/ASIC attacks per current OWASP guidance.
- ADR-046: JWT lightweight design — `tenant.jwt_active_token` stores the currently-valid token (HS256, 24h TTL). Middleware accepts either `Authorization: Bearer <jwt>` or `X-Tenant-API-Key: aitkey_…`; on JWT decode failure or mismatch with `jwt_active_token`, falls back to API key auth. MVP-grade (single active token, no rotation cron); production-grade JWT key management deferred.
- ADR-047: New frontend (Vite + React + TS + Tailwind + shadcn/ui + Framer Motion) replaces Streamlit demo `demo/app.py`. Reason: design brief minta visual rich + animasi tinggi (SVG dot-flow di pipeline, typewriter streaming, parallel completion viz, mismatch banner shake) yang Streamlit batasi karena rerun-cycle model. Demo audience teknis butuh polish stakeholder-grade. Streamlit `demo/app.py` dihapus setelah React frontend verified end-to-end. `demo/webpage/` Phase 7 SDK landing page RETAINED (unrelated to operator UI).
- ADR-048: Frontend TypeScript types match real sub-proyek I `/translate` response shape (canonical), bukan brief's illustrative payload. `services/types.ts` declares `TranslateResponse`, `AgenticActivity`, `TranslateApi` mirroring backend. Reason: cheap mock-to-real swap (single file replacement at integration phase — `realApi.ts` implements same `TranslateApi` interface). Brief's payload example tetap dijadikan visual reference untuk highlighting/layout di PayloadViewer.
- ADR-049: Tab 1 (Tenant Management) di frontend mock-only forever. Brief's tenant model (name + langs + model_tier + 3 toggles) sengaja simpler dari sub-proyek I tenant junction (country/company/department × position × service). Real cascade UI deferred ke future sub-project — UX needs berbeda (multi-step wizard + dynamic dropdown chaining) dan demo audience focus consumer-app feel, not operator config form. Tenant CRUD purely in-memory `useState<Tenant[]>` di App.tsx.
- ADR-050: PowerShell launcher (`scripts/run-demo.ps1`) force-open Chrome explicitly (bukan default browser). Reason: user requirement, predictable behavior. Resolves chrome.exe via `$env:CHROME_PATH` + `Get-Command chrome.exe` + 3 standard install paths (Program Files, Program Files (x86), LocalAppData\Google\Chrome). Vite TIDAK pakai `--open` flag — launcher owns browser open via `Start-Job` polling `localhost:5173` + `Start-Process chrome.exe`. Foreground `npm run dev` so Vite logs visible.
- ADR-051: shadcn/ui adopted sebagai base primitive layer (Button, Card, Tabs, Select, Badge, Dialog, Tooltip) bukan Material/Chakra/Mantine. Reason: copy-paste primitives accessible by default (Radix UI under the hood), no runtime CSS-in-JS, generated components tinggal di repo (`src/components/ui/`, versioned, customizable in-place). Trade-off: lebih banyak files vs single npm import — acceptable for a demo-grade scaffold yang akan grow ke real-integration phase.
- ADR-052: Frontend branding aligned to **PT. Integrity Indonesia identity** — "AI Translation by Integrity" (not "Polyglot AI" from initial brief). Palette swap from blue (cyan #06b6d4 + violet #8b5cf6) to **merah-putih** (Indonesian flag-inspired): `accent-red #B91C1C` for primary brand + CTAs, `accent-rose #F43F5E` for gradient pair, `accent-amber #F59E0B` for warnings (mismatch banner — shifted FROM crimson to prevent conflict with brand red), `accent-emerald #10B981` for success, `accent-crimson #EF4444` reserved for true error/failure states. Logo is geometric owl head silhouette (Integrity's mascot/trademark) — inline SVG in TopBar + standalone in `public/favicon.svg`. Reason: brand identity match for stakeholder demos; merah-putih + owl carry the company's visual language. Implementation in commit `a4a9e6e` after initial sub-proyek J commit `257ec4e`.
- ADR-053: Tenant + tenant_profile denormalize ke snapshot name columns (drop FK to country/company/department/position/service reference tables). Catalog tables retained untuk cascade UI + Jinja context lookup by-name. Trade-off: audit-stable rename-doesn't-propagate vs FK-integrity loss. Snapshot preferred untuk translation audit + log row self-containment.
- ADR-054: `alembic_version_at_create VARCHAR(60) NOT NULL` snapshot column di tenant. Set saat seed/INSERT dengan baca Postgres `alembic_version` meta-table. Audit-cohort tracking ("tenant ini dibuat saat schema versi X"). Tidak auto-updated kalau migration baru jalan — existing rows tetap pegang versi creation lama.
- ADR-055: `prompt_applied` stored sebagai array of `agent_type` strings (bukan `prompt_id`). Karena `tenant_prompts.agent_type` UNIQUE, informationally equivalent tapi jauh lebih readable di DB inspection + grep. DB-level CHECK enforces length = 3; ordering enforced di Pydantic validator (`EXPECTED_PROMPT_APPLIED_ORDER`).
- ADR-056: `allowed_language` stratified deterministic distribution across 5 patterns (12/12/11/11/11 dari 57 tenant_profiles): `[id,en]` / `[ms,en]` / `[th,en]` / `[id,ms,th,en]` / NULL=all. Ordering by `(company_name, department_name)` deterministic. Pipeline reject `target_lang ∉ allowed_language` dengan `LanguageNotAllowedError → HTTP 400 error_code = language_not_allowed`. Failed request tetap create log row dengan `status = failed` per ADR-027.
- ADR-057: `iso_languages` lookup module-level in-memory cache (~40 rows), populated on first call (`IsoLanguageRepository._catalog_cache`). Process-restart invalidates. Fallback: code as-is + log warning kalau miss. Tidak perlu Redis — table kecil + read-only seed fixture.
- ADR-058: Single flat Jinja context dict shared across 3 prompt templates (`lang_detect_input`, `translate`, `lang_detect_output`). Variables sourced dari `ResolvedTenantProfile` dataclass (returned by refactored `TenantProfileResolver`). Template author decides per-agent relevance — lang_detect templates can ignore service-specific vars. Source of truth: `src.pipeline.stages.build_jinja_context`.
- ADR-059: Frontend adapter pattern over types.ts canonical rewrite (sub-proyek L). `frontend-demo/src/services/responseAdapter.ts` maps backend `TranslateResponse` → frontend `TranslateResponse` without forcing alignment. Components stay unchanged; future backend swap edits adapter, not types. Trade-off: type-system not the single source of truth across stack.
- ADR-060: Synthetic streaming for agentic events. Real `/translate` is one-shot; frontend `realApi.ts` replays `agent_completed` events spaced by `activity.latency_ms` to preserve `AgentPipeline` progressive-parallel animation. Cache hit short-circuits replay (all events fire immediately). Future SSE/WebSocket replaces the replay loop.
- ADR-061: Settings modal route, NOT Tab 1 tenant embedding. Backend credentials (api_key + tenant_id + profile_id + baseUrl) configured separately from mock tenant management. Honors ADR-049 "Tab 1 mock-only forever". Trade-off: two unlinked entities (mock tenant vs real credentials); mitigated by gear-icon framing + explicit modal copy ("Backend Connection / Currently using: REAL backend").
- ADR-062: `VITE_API_MODE` env var toggle, default `mock`. Demo runs out-of-box without backend; operator sets `=real` in `.env.local` (gitignored) when ready. `.env.local.example` committed as template. No runtime UI toggle in MVP — `apiSelector.ts` re-reads localStorage settings every call so Settings Save takes effect without page reload.
- ADR-063: Vite dev proxy `/api/*` → `http://localhost:8000`. Frontend code uses relative paths (`/api/translate`). Backend CORS allowlist extended to `localhost:5173` for direct-hit fallback (when user overrides baseUrl to absolute URL in Settings). Production deployment uses reverse proxy with identical path mapping.
- ADR-064: Explicit `error_code` → UI banner mapping (`language_not_allowed` → amber; `missing_credentials`/`provider_auth_failed` + forward-compat `authentication_failed`/`tenant_not_found` → crimson; `rate_limited` → amber; `upstream_transient` → crimson; generic/network → crimson). Banner positioned ABOVE OutputBox, severity-tinted per ADR-052 palette. Copy-on-click `trace_id` for support correlation. Auto-dismiss on next successful translate. The auth bucket covers what backend actually emits (`missing_credentials` from `src/auth/middleware.py` when no header is present; `provider_auth_failed` from `src/api/middleware.py` when upstream provider auth fails); the other two codes are reserved slots for future backend changes so the frontend predicate doesn't churn on each rename.
